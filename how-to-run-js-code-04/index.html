JavaScript Variables Example: var, let, const

Code Explanation

var firstName = 'Akash';
The firstName variable is declared using var, which is hoisted to the top of its scope and initialized as undefined in the memory allocation phase. Afterward, during the execution phase, it is assigned the value 'Akash'.

let lastName = 'Singh';
The lastName variable is declared using let. It is hoisted but remains uninitialized in the memory allocation phase (it enters the Temporal Dead Zone). The assignment happens during execution, where it gets the value 'Singh'.

let age = 15;
Similar to lastName, the age variable is hoisted but uninitialized until it reaches this line during execution. It is then assigned the value 15.

const yearOfBirth = 1999;
The yearOfBirth variable is declared with const, meaning it must be initialized at the time of declaration and cannot be reassigned later. Like let, it is hoisted but not initialized until execution, and attempting to access it beforehand would cause a ReferenceError.

String Concatenation:
let userIntro = 'Hi, my name is ' + firstName + ' ' + lastName;
This line concatenates the strings and variables firstName and lastName to form the sentence: 'Hi, my name is Akash Singh'.

In JavaScript, the Temporal Dead Zone (TDZ) is the area of a block where a variable declared with let or const exists but cannot be accessed until it has been initialized.
