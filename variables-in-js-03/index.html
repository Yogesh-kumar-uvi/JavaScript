Variables in JavaScript

A JavaScript variable is just a name of storage location.

And we can give that name in three ways.

  • Using let keyword.
  • Using const keyword.
  • Using var keyword.

Difference between let, const, and var

var

  • Function-scoped: It's accessible within the function where it's declared.
  • Hoisting: Can be used even before it's declared (but it's initialized as undefined).
  • Re-declaration allowed: You can declare the same variable multiple times.

let

  • Block-scoped: Only accessible within the block {} where it's declared.
  • No hoisting: Cannot be used before declaration.
  • No re-declaration: You cannot declare the same variable twice in the same scope.

const

  • Block-scoped: Same as let, only within the block {}.
  • Constant value: Once assigned, it cannot be reassigned (but objects or arrays can be modified).
  • No hoisting and No re-declaration.

JavaScript Variable Declaration Rules

1. Use var, let, or const for Variable Declarations

Always use var, let, or const to declare variables. If you don't, JavaScript will create a global variable, which can lead to errors.

var x = 10;    // OK
let y = 20;    // OK
const z = 30;  // OK

2. Variable Names

  • Must start with a letter, underscore (_), or dollar sign ($).
  • Cannot start with a number.
  • Variable names are case-sensitive (e.g., myVar and myvar are different).
  • Avoid using reserved words (like if, for, class).
let _myVar = 5;  // OK
let $money = 100;  // OK
let 1stPlace = 3;  // Error: Cannot start with a number

3. Avoid Re-declaring with let and const

You cannot re-declare a variable using let or const in the same scope. var allows re-declaration, but it's better to avoid it to prevent confusion.

let a = 10;
// let a = 20;  // Error: Cannot re-declare

var b = 5;
var b = 10;  // Allowed but should be avoided

4. const Requires Initialization

Variables declared with const must be initialized at the time of declaration, and once assigned, they cannot be reassigned.

const PI = 3.14;
// PI = 3.14159;  // Error: Cannot reassign

const x;  // Error: Missing initialization

5. Block Scope for let and const

let and const are block-scoped, meaning they are only accessible within the block {} where they are declared. var is function-scoped, meaning it is accessible throughout the function.

if (true) {
    let blockVar = "I'm block-scoped";
    var funcVar = "I'm function-scoped";
}

// console.log(blockVar);  // Error: blockVar is not defined
console.log(funcVar);  // OK: "I'm function-scoped"

6. Hoisting

var is hoisted to the top of its scope and initialized as undefined. let and const are hoisted but not initialized, so they cannot be used before being declared.

console.log(x);  // undefined (due to var hoisting)
var x = 10;

console.log(y);  // Error: Cannot access 'y' before initialization
let y = 20;

7. Best Practice for Declaration

  • Use const for variables that should not change.
  • Use let for variables that will be reassigned.
  • Avoid using var unless you're working with older or legacy code.
